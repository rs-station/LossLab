# LossLab SAXS Losses: Design and API Reference

## Overview

LossLab provides three differentiable SAXS loss functions for protein structure refinement,
all based on the Debye scattering equation. They compute I(q) from atomic coordinates and
compare to experimental SAXS profiles, with gradients flowing back through the coordinates.

Source: `LossLab/losses/saxs.py`

## Loss Classes

### 1. `DebyeRawLoss` — Effective-Charge Weighted Debye

The simplest loss. Computes:

```
I(q) = sum_ij w_i * w_j * sin(q*r_ij) / (q*r_ij)
```

where `w_i = f_i(0) + n_implicit_H_i` is the effective charge (number of electrons
including implicit hydrogens). No q-dependent form factors, no excluded volume.

**When to use:** Quick sanity checks, matching pyAUSAXS `mol.debye_raw()`.

**Parameters:**
- `ff_types`: list of atom type strings (e.g., `["C", "CH", "NH", "O"]`); `None` for uniform weights
- `scale_invariant`: if True, fits a log-space offset before computing chi2

---

### 2. `DebyeLoss` — Full Debye with Form Factors and ExV

The standard SAXS loss with q-dependent 5-Gaussian form factors and optional Fraser
excluded volume correction:

```
f_i(q) = sum_k a_k * exp(-b_k * q^2) + c        # 5-Gaussian parameterisation
f_i^exv(q) = V_i * rho_0 * exp(-V_i^(2/3) / (4*pi) * q^2)   # Fraser ExV
f_i^eff(q) = f_i(q) - cx * f_i^exv(q)            # effective form factor

I(q) = sum_ij f_i^eff(q) * f_j^eff(q) * sin(q*r_ij) / (q*r_ij)
```

**When to use:** Atom-only SAXS refinement (no hydration shell). This is the protein-only
Debye equation equivalent to CRYSOL/FoXS without the hydration term.

**Key parameters:**
- `use_exv` (default True): Apply Fraser excluded volume correction. When False, uses
  effective-charge-weighted form factors instead (matches pyAUSAXS `ExvModel.none`).
- `match_ausaxs` (default False): Replicate the pyAUSAXS AX asymmetry bug for validation.
  **Never use in production — always False for training.**
- `chi2_mode`: `"log"` (default) or `"linear"` (standard SAXS chi2 with optimal scale factor)
- `detach_scale`: When True with `chi2_mode="linear"`, the scale factor `c` is detached from
  the computation graph. This prevents the optimizer from gaming the scale — it can only
  reduce chi2 by improving the structural fit. Mirrors CRYSOL/FoXS refinement practice.
- `q_chunk_size`: Process q-values in chunks to bound GPU memory at O(chunk * N^2) instead of O(Q * N^2).

**Design choice — two form factor conventions:**

| `use_exv` | Form factor | Convention | Matches |
|-----------|------------|------------|---------|
| `False` | `f(q) * (f(0) + n_H) / f(0)` | Effective-charge weighted | pyAUSAXS `ExvModel.none` |
| `True` | `f(q) - f_exv(q)` | Raw 5-Gaussian minus ExV | CRYSOL/FoXS/Fraser |

When `use_exv=True`, the raw form factor is used (not effective-charge weighted) because the
excluded volume correction already accounts for the solvent displacement. Multiplying by
effective charge would double-count the implicit hydrogen contribution.

---

### 3. `DebyeHydrationLoss` — Three-Component Debye with Hydration Shell

The full SAXS loss including the hydration shell contribution:

```
I(q) = I_aa(cx) + 2 * cw * I_aw(cx) + cw^2 * I_ww

where:
  I_aa = sum_ij f_i^eff * f_j^eff * sinc(q*r_ij)    # atom-atom
  I_aw = sum_ij f_i^eff * f_w_j  * sinc(q*r_ij)     # atom-water cross
  I_ww = sum_ij f_w_i  * f_w_j   * sinc(q*r_ij)     # water-water
```

Atom form factors use Fraser ExV (`f_eff = f_raw - cx * f_exv`).
Water form factors use raw 5-Gaussian (no ExV — shell is excess density above bulk).

**Key parameters:**
- `cx` (default 1.0): Excluded volume scale factor. Controls how much displaced solvent to subtract.
- `cw` (default 1.0): Hydration shell contrast. Scales the water contribution.
- `learn_cx`, `learn_cw`: Make cx/cw learnable `nn.Parameter`s optimized during training.
- `water_mode`: `"fixed"` or `"attached"` (see below).

**Water modes:**

| Mode | Waters move? | Gradient through waters? | I_ww cached? | Use case |
|------|-------------|------------------------|-------------|----------|
| `fixed` | No | No (atom-side of I_aw only) | Yes (constant) | Fast; good when structure moves little |
| `attached` | Yes (with nearest atom) | Yes (through offsets) | No | Full gradient; captures shell deformation |

In `attached` mode, each water is assigned to its nearest protein atom via `torch.cdist`.
A frozen offset `delta = water_pos - atom_pos[nearest]` is stored. When atoms move,
`water_pos = atom_pos[nearest] + delta`, so gradients flow through both the atom positions
and the water cross-term.

**Rehydration:** When coordinates drift significantly (>2-3 A RMSD), the hydration shell should
be regenerated by calling `set_water()` again with fresh waters from pyAUSAXS `mol.hydrate()`.
In the of3compass wrapper, this is controlled by `--saxs-rehydrate-every N`.

## Standalone Functions

### `debye_intensity(coords, ff_types, q_values, use_exv, cx, rho_water)`

Stateless function computing I(q) from coordinates. Same physics as `DebyeLoss._I_pred()`.
Useful for one-off evaluations without creating a loss object.

### `debye_hydration_intensity(coords_atom, ff_types_atom, coords_water, ff_types_water, q_values, cx, cw)`

Stateless three-component Debye. Returns `(I_total, {"I_aa": ..., "I_aw": ..., "I_ww": ...})`.
Always uses Fraser ExV on atom form factors.

## Chi2 Modes

### Log-space chi2 (`chi2_mode="log"`, default)

```
chi2 = mean( (log(I_pred) - log(I_target))^2 )
```

With `scale_invariant=True`, a detached mean offset is applied in log-space before computing
residuals, making the loss invariant to an overall multiplicative scale.

**Pros:** Treats all q-points equally in log-space; robust to dynamic range.
**Cons:** Not the standard SAXS metric reported in literature.

### Linear-space chi2 (`chi2_mode="linear"`)

```
c = sum(I_exp * I_calc / sigma^2) / sum(I_calc^2 / sigma^2)
chi2 = (1/N) * sum( ((I_exp - c * I_calc) / sigma)^2 )
```

The analytically optimal scale factor `c` is computed at each step. When `detach_scale=True`,
gradients do not flow through `c`.

**Pros:** Standard SAXS metric (matches CRYSOL, FoXS, pyAUSAXS `mol.fit()`); sigma-weighted.
**Cons:** Dominated by low-q points (where sigma is smallest and I is largest).

**Recommendation for refinement:** Use `chi2_mode="linear"` with `detach_scale=True`. This is the
standard in structural biology and prevents scale-factor gaming.

## Form Factor Types

Atom types follow the pyAUSAXS/AUSAXS convention with implicit hydrogens:

| Type | Description | Implicit H | Example atoms |
|------|------------|-----------|---------------|
| `C` | Carbon, no H | 0 | Backbone C, ring C in PHE/TRP |
| `CH` | Carbon, 1H | 1 | CA (non-GLY), aromatic CH |
| `CH2` | Carbon, 2H | 2 | GLY CA, CB of most residues |
| `CH3` | Carbon, 3H | 3 | ALA CB, methyl groups |
| `N` | Nitrogen, no H | 0 | PRO N, ring N |
| `NH` | Nitrogen, 1H | 1 | Backbone N (non-PRO), HIS ring NH |
| `NH2` | Nitrogen, 2H | 2 | ASN ND2, ARG NH1/NH2 |
| `NH3` | Nitrogen, 3H | 3 | LYS NZ |
| `O` | Oxygen, no H | 0 | Backbone O, ASP OD1, GLU OE1 |
| `OH` | Oxygen, 1H | 1 | SER OG, THR OG1, ASP OD2, GLU OE2, water |
| `S` | Sulfur, no H | 0 | MET SD |
| `SH` | Sulfur, 1H | 1 | CYS SG |

The mapping from (residue_name, atom_name) to form factor type is done by `assign_ff_type()`
in `of3compass/losses/saxs.py`. This was validated to produce 0 mismatches against pyAUSAXS's
internal type assignment on NFU1 (1375 atoms).

**Notable assignments:**
- GLY CA -> CH2 (not CH; glycine has 2 hydrogens on C-alpha)
- ASP OD2 -> OH (protonated carboxylate oxygen)
- GLU OE2 -> OH (same)
- OXT -> OH (C-terminal oxygen)

## Memory and Performance

The full Debye sum is O(Q * N^2) in compute and O(Q * N^2) or O(N^2) in memory (depending
on chunking). For typical proteins:

| N atoms | Q points | q_chunk_size | Peak GPU memory |
|---------|----------|-------------|----------------|
| 1000 | 200 | None | ~1.5 GB |
| 1000 | 200 | 16 | ~120 MB |
| 2000 | 1774 | 16 | ~500 MB |

Use `q_chunk_size=16` (default in of3compass) for proteins >500 atoms. The chunks iterate
over q-values, materialising the (chunk, N, N) sinc tensor per chunk and concatenating results.
Autograd-safe (uses `torch.cat`, not in-place assignment).
